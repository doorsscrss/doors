-- Doors Script Configuration File
-- Save as: config.lua in your scripts directory

local DoorsConfig = {

## 🛠️ Troubleshooting Common Issues

**❌ Script does not execute after clicking the button**
The tool may not be properly attached to the game process. Make sure the game is fully loaded past the main menu before clicking **Attach**. If the issue persists, restart both the game and the tool, then try again.
**Solution:** Close your tool completely, relaunch it, wait for the game lobby to fully render, then attach and execute the **Doors Script** in that order.

**❌ GUI panel does not appear in-game**
This typically happens when another script's GUI is conflicting or when `showHUD` is set to `false` in the config file.
**Solution:** Check your `config.lua` and confirm `showHUD = true`. If another script was loaded previously, rejoin the server with a fresh session before executing the **Roblox Doors Script**.

**❌ Auto Farm stops after a few loops**
Extended farming can trigger a session timeout or a memory overflow in older tool versions.
**Solution:** Update your tool to the latest stable release, set `autoRejoin = true` in the config, and reduce `speedMultiplier` to `1.0` during farm sessions to lower resource consumption by the **Auto Farm Doors Script**.

**❌ Badges not unlocking despite module being active**
Some badge conditions require specific room interactions that the **Doors Script All Badges** module may miss if `autoSkip` is simultaneously enabled.
**Solution:** Disable `autoSkip` when running the badge collection module. The skip function bypasses rooms that contain required badge triggers for the **Best Doors Script** to detect.

**❌ Entity bypass fails on specific entities**
Certain entities like Figure require unique avoidance logic that differs from Rush or Ambush handling.
**Solution:** Make sure you are running version `2.8.1` or later — earlier builds had incomplete Figure detection. If updated and still failing, enable `debugMode` and check the log output for entity ID mismatches.
